# AI 配置文件夹

本文件夹包含所有与 AI 编码助手相关的配置文件。

## 文件说明

### `AGENTS.md`
项目的代码规范与开发指南，供 AI 编码助手（如 OpenCode）使用。包含：
- 构建/测试命令
- 代码风格指南
- 命名约定
- 项目结构说明

**应该提交到 Git**：是 ✅

### `opencode.json`
OpenCode 的项目级配置文件，包含项目特定的权限设置。

**应该提交到 Git**：是 ✅

### 其他文件
临时文件、会话记录等不应提交到 Git 的文件会被 `.gitignore` 自动忽略。

## 权限配置架构

本项目采用**两层权限配置**：

### 1. 全局配置
位置：`~/.config/opencode/opencode.json`

包含所有项目通用的权限规则：
- 基础文件读写权限
- Git 命令权限
- NPM/包管理器权限
- 系统命令权限
- 网络请求权限

### 2. 项目配置
位置：`.ai/opencode.json`（本文件夹）

包含本项目特定的权限规则：
- 样式文件自动允许修改
- TypeScript/React 文件需要询问
- 项目配置文件需要询问
- AI 配置文件夹内文件允许修改

## 配置优先级

项目配置会**覆盖**全局配置中的同名规则。例如：
- 全局配置：`*.css` 需要询问
- 项目配置：`*.css` 自动允许
- 最终效果：本项目中修改 CSS 文件会自动允许

## 使用建议

1. **全局配置**适合放置：
   - 安全相关的规则（拒绝危险命令）
   - 所有项目通用的规则（Git 查询命令）
   - 默认行为设置

2. **项目配置**适合放置：
   - 项目特定的文件类型权限
   - 项目特定的构建命令
   - 团队协作的权限约定

## 修改配置

修改配置后需要重启 OpenCode 才能生效。

```bash
# 编辑全局配置
code ~/.config/opencode/opencode.json

# 编辑项目配置
code .ai/opencode.json

# 重启 OpenCode
# 退出当前会话，重新运行 opencode 命令
```
