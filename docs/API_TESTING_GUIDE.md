# API 接口接入测试指南

## 构建验证

✅ **构建成功** - 项目编译通过，没有 TypeScript 错误

## 手动测试步骤

### 1. 启动开发服务器

```bash
npm run dev
```

### 2. 测试登录功能

1. 访问 `/login` 页面
2. 输入测试账号（根据接口文档）：
   - 用户名: `wrj1` 或 `zzz`
   - 密码: `123456`（将被自动 MD5 加密）
3. 点击登录

**预期结果：**
- 密码被 MD5 加密为 `21232f297a57a5a743894a0e4a801fc3`
- 调用 `POST /admin/login` 接口
- 成功后调用 `GET /workPlatform/list` 获取任务信息
- 跳转到 `/dashboard` 页面
- 显示正确的用户信息、队伍信息和任务信息

### 3. 测试任务信息显示

在 Dashboard 页面检查：
- ✓ 用户名和 ID 正确
- ✓ 队伍名称和 ID 正确
- ✓ 任务名称和描述正确
- ✓ 角色根据 taskRoleId 正确显示：
  - taskRoleId = "2" → 定位分析
  - taskRoleId = "3" → 路径规划

### 4. 测试登出功能

1. 点击"退出登录"按钮
2. 观察网络请求（F12 开发者工具）

**预期结果：**
- 调用 `POST /admin/logout` 接口
- 清除本地 token 和用户信息
- 跳转到 `/login` 页面

### 5. 测试错误处理

**测试错误凭证：**
1. 输入错误的用户名或密码
2. 观察错误提示

**预期结果：** 显示"用户名或密码错误"

**测试 token 过期：**
1. 手动清除 localStorage 中的 token
2. 刷新页面或访问需要认证的页面

**预期结果：** 自动跳转到登录页

### 6. 测试网络错误处理

**模拟服务器不可用：**
1. 确保后端服务未启动
2. 尝试登录

**预期结果：** 显示"网络连接失败，请检查网络设置"

## 环境配置

确保 `.env.development` 配置正确：

```env
VITE_API_BASE_URL=http://localhost:28888
VITE_USE_MOCK=false
```

## 切换到 Mock 模式

如果需要使用 Mock 数据进行测试，修改 `.env.development`：

```env
VITE_USE_MOCK=true
```

## 注意事项

1. **后端服务**: 确保后端服务运行在 `http://localhost:28888`
2. **密码加密**: 密码自动进行 MD5 加密
3. **Token 格式**: 使用 `Bearer ${token}` 格式
4. **时间格式**: 任务时间从数组格式自动转换为 Date 对象
5. **角色判断**: taskRoleId 为字符串类型，已正确处理
