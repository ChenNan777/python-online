# 路网配置页面开发计划

## 1. 目标

创建一个路网配置页面，用于可视化展示和编辑路网数据。该页面将作为“练习模式”的一部分入口，允许用户查看和修改路网属性（目前仅前端状态，暂无后端保存接口）。

## 2. 路由与入口

* **新路由**: `/road-network-config` (暂定)

* **入口位置**: 在练习模式页面（`PracticePage` 或类似位置）增加“路网配置”入口按钮。

## 3. 页面布局与功能

### 3.1 整体布局

* **全屏地图**: 使用 Leaflet (React-Leaflet) 渲染，保持与考试页面一致的视觉风格。

* **右侧配置面板**: 当选中路段时，右侧弹出属性配置面板（Drawer 或 Sider），展示该路段的详细信息。

### 3.2 地图交互

* **加载数据**: 复用 `useExamRoadDataQuery` 获取 `/StudentOperationCode/road-data` 数据。

* **渲染路网**:

  * 解析 GeoJSON 数据。

  * 绘制路段（Polyline）和节点（CircleMarker）。

  * 支持缩放、平移。

* **选中交互**:

  * 点击路段高亮显示（例如变为亮黄色）。

  * 点击空白处取消选中。

### 3.3 属性配置

* **展示字段**:

  * 路段 ID (Road ID)

  * &#x20;长度 (Length)

  - 权重 (Weight) - *可编辑*

  - ~~车道数 (Lanes) - 暂时隐藏~~

  - ~~最大速度 (Max Speed) - 暂时隐藏~~

  - ~~拥堵系数 (Congestion) - 暂时隐藏~~

* **编辑行为**:

  * 修改后实时更新本地状态（暂不持久化到后端）。

  * 提供“重置”按钮恢复默认值。

## 4. 技术方案

* **状态管理**: 使用 `useState` 或 `useReducer` 管理当前加载的路网数据及选中状态。考虑到无后端保存，编辑后的数据仅在当前页面生命周期内有效。

* **组件复用**:

  * 新建 `RoadNetworkMap`。

  * 复用 `src/utils/parseRoadNetwork.ts` 进行数据解析。

## 5. 开发任务拆解

1. [ ] **路由配置**: 在 `routes.ts` 注册 `/road-network-config`。
2. [ ] **页面骨架**: 创建 `src/pages/RoadNetworkConfigPage`，包含地图容器和侧边栏结构。
3. [ ] **数据加载**: 对接 `useExamRoadDataQuery`，实现 GeoJSON 解析与渲染。
4. [ ] **地图交互**: 实现路段点击选中、高亮样式。
5. [ ] **属性面板**: 开发右侧配置表单，支持属性编辑。
6. [ ] **入口添加**: 在练习模式页面增加跳转入口。

## 6. 验证计划

* 确认页面能正常加载路网数据。

* 确认点击路段能正确弹出面板并显示对应属性。

* 确认修改属性后，地图或面板上的显示能同步更新（如需可视化反馈）。

* 确认全流程无 TypeScript 错误，构建通过。
