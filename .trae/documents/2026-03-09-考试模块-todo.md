# 考试模块 TODO（逐项推进）

范围：考试链路（`ExamChallengePage` → `ChallengeWorkspace` → 运行/保存/提交 → 历史/场景）

## 接口字段对齐清单（先对齐再过逻辑）

- [ ] 获取作业信息：GET `/StudentOperationCode/assignment-info`
  - 入参：`userId: number`
  - 回参：`StudentTrainingAssignmentVO`
    - 页面/提交强依赖：`taskId`、`memberId`、`teamId`（保存/提交/拉历史）
    - 提交定位结果依赖：`targetId: string`（提交时需要转成 `uavId: number`，并校验可转）
    - 提交路径规划依赖：`roadNetworkGeoJson`、`startLongitude`、`startLatitude`、`targetLongitude`、`targetLatitude`
    - 提交接口附带：`memberName`（可选，但当前实现会带上）

- [ ] 获取历史代码：GET `/StudentOperationCode/list/student/{taskId}/{memberId}`
  - 入参：`taskId: number`、`memberId: number`
  - 回参：`StudentOperationCodeVo[]`
    - 页面使用字段：`sourceCode`（初始化/恢复）、`updatedAt/submitTime/createdAt`（排序/取最新）

- [ ] 保存代码：POST `/StudentOperationCode/saveCode`
  - 入参：`StudentOperationCodeDTO`
    - 页面发送字段：`taskId`、`memberId`、`teamId`、`operationType('1'|'2')`、`programmingLanguage('Python')`、`sourceCode`
  - 回参：`boolean`（当前页面以本地 `lastSavedCode` 为准，不依赖回参）

- [ ] 提交作业：POST `/StudentOperationCode/submit`
  - 入参：`StudentTrainingWorkSubmitDTO`
    - 共用字段：`taskId`、`teamId`、`memberId`、`memberName?`、`workType(4|5)`
    - 定位题（`workType=4`）：`positioningData: { uavId:number; longitude:number; latitude:number; accuracy?; analysisDescription? }`
    - 路径规划题（`workType=5`）：`pathPlanningData: { planningResult: { pathId:string; algorithmUsed:string; totalDistance?; estimatedDuration?; waypoints:{longitude;latitude;sequence?}[] } }`
  - 回参：`string`（当前页面只展示“提交成功/失败”，不做解析）

- [ ] 定位场景（待后端接口）：当前仅 fallback
  - 触发条件：仅定位题 + `assignment.targetId` 存在时才会走 query
  - 当前实现：本地样例场景 + `targetId` 覆盖目标身份，并展示 fallback 提示
